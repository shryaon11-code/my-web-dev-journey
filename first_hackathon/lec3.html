Welcome to Lecture 3!

So far, you’ve learned about the basics of HTML, text formatting, links, and images. Now it’s time to dive into two powerful structures:

  • Lists -- Great for organizing content, menus, or even step-by-step instructions.
  • Tables — Perfect for presenting structured data like schedules, product details, or comparison charts.

This lecture will show you how to use lists (including nested lists) and tables, while teaching you real-world best practices.

Lists in HTML

Lists help display information in a clean, organized way.

  • Ordered Lists <ol> -- Numbered lists (1, 2, 3 …).
  • Unordered Lists <ul> -- Bulleted lists.
  • List Item <li> -- Each item inside the list.

Example:

<ul>
<li>Fruits</li>
<li>Vegetables</li>
</li>

<ol>
<li>Step 1: Open editor</li>
<li>Step 2: Write HTML</li>
</ol>

Nested Lists

You can place one list inside another to create hierarchies.

Example:

<ul>
<li>Fruits
<ul>
<li>Apple</li>
<li>Banana</li>
</ul>
</li>
<li>Vegetables
<ul>
<li>Carrot</li>
<li>Potato</li>
</ul>
</li>
</ul>

Use Cases:

  • Website navigation menus
  • Course outlines
  • FAQ categories

Tables in HTML

Tables arrange data in rows and columns.

Basic Structure:

  • <table> -- Creates the table
  • <tr> -- Table row
  • <td> -- Table data cell
  • <th> -- Table header cell

Example:

<table border="1">
<tr>
<th>Name</th>
<th>Age</th>
</tr>
<tr>
<td>Froggy</td>
<td>5</td>
</tr>
</table>

Table Sections

To make tables more structured:

  • <thead> -- Table header
  • <tbody> -- Table body
  • <tfoot> -- Table footer

Example:

<ul border="1">
<thead>
<tr>
<th>Product</th>
<th>Price</th>
</tr>
</head>
<tbody>
<tr>
<td>Notebook</td>
<td>$5</td>
</tr>
</tbody>
<tfoot>
<tr>
<td>Total</td>
<td>$5</td>
</tr>
</tfoot>
</table>

Advanced Table Features

  • colspan -- Merge columns
  • rowspan -- Merge rows

Example:

<table border="1">
<tr>
<th rowspan="2">Name</th>
<th colspan="2">Scores</th>
</tr>
<tr>
<td>Math</td>
<td>Science</td>
</tr>
<tr>
<td>Lily</td>
<td>90</td>
<td>95</td>
</tr>
</table>

Pro Tips

  • Always use alt text for images in nested lists or menus (for accessibility).
  • Avoid very large tables — they are harder to read on small screens.
  • Keep table design minimal when using HTML only. Later, with CSS, we can make them prettier.
  • Nested lists are great for menus, but don’t nest too deeply (it confuses users).
